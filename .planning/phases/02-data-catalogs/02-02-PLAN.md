**Model (backend/app/models/client.py):**
```python
class Client(Base):
    __tablename__ = "clients"

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)

    # Identification — DIAN document type codes (Tabla 13.2.1)
    document_type: Mapped[str] = mapped_column(String(5), nullable=False)
    # Valid codes: 11,12,13,21,22,31,41,42,47,48,50,91
    document_number: Mapped[str] = mapped_column(String(20), nullable=False)
    check_digit: Mapped[str | None] = mapped_column(String(1), nullable=True)
    # DV required only when document_type == "31" (NIT)

    # Name
    business_name: Mapped[str] = mapped_column(String(255), nullable=False)
    trade_name: Mapped[str | None] = mapped_column(String(255), nullable=True)

    # Contact
    email: Mapped[str] = mapped_column(String(255), nullable=False)
    phone: Mapped[str | None] = mapped_column(String(30), nullable=True)

    # Address
    address_line: Mapped[str] = mapped_column(Text, nullable=False)
    department_code: Mapped[str] = mapped_column(String(2), nullable=False)
    municipality_code: Mapped[str] = mapped_column(String(5), nullable=False)
    postal_code: Mapped[str | None] = mapped_column(String(10), nullable=True)

    # Fiscal — DIAN codes
    fiscal_responsibilities: Mapped[str] = mapped_column(String(50), nullable=False, default="R-99-PN")
    # Semicolon-separated: "O-13;O-15" or single "R-99-PN"
    tax_regime: Mapped[str] = mapped_column(String(10), nullable=False, default="49")
    # 04=Simplificado, 05=Ordinario, 48=IVA Responsable, 49=No responsable

    is_active: Mapped[bool] = mapped_column(Boolean, default=True)
```

Add a unique constraint on `(document_type, document_number)` — same document number can exist for different document types.

**Schemas (backend/app/schemas/client.py):**
- `ClientCreate` — all required fields, optional fields with defaults. Add `@model_validator(mode="after")` that:
  - Validates document_type is in the allowed set: {"11","12","13","21","22","31","41","42","47","48","50","91"}
  - If document_type == "31" (NIT), requires check_digit and validates it using `nit.calculate_check_digit()`
  - Validates fiscal_responsibilities contains only valid codes separated by semicolons
  - Validates tax_regime is in {"04","05","48","49"}
- `ClientUpdate` — same as Create (for PUT replacement)
- `ClientResponse` — includes `id` and `is_active`, with `model_config = {"from_attributes": True}`

**Router (backend/app/routers/clients.py):**
`APIRouter(prefix="/api/clients", tags=["clients"])`

Endpoints following exact same pattern as resolutions.py:
1. `GET /api/clients` → list all active clients, ordered by business_name. Support optional `q: str` query param for search (ILIKE on business_name or document_number).
2. `POST /api/clients` → create client. Return 201.
3. `GET /api/clients/{id}` → get single client. 404 if not found.
4. `PUT /api/clients/{id}` → update client.
5. `DELETE /api/clients/{id}` → soft delete (set is_active=False). Return 204.

All endpoints require `Depends(get_db)` and `Depends(get_current_user)`.
