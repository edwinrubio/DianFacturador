# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation and Setup.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope: Login page, 4-step onboarding wizard, dashboard stub.
> Language: Spanish throughout (all UI copy, labels, messages).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | none — initialize with `npx shadcn@latest init` during Phase 1 implementation |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react (ships with shadcn/ui) |
| Font | Inter (system fallback: ui-sans-serif, system-ui, sans-serif) |

**shadcn components required for this phase:**
- `button` — all CTAs and form submission
- `card` — wizard step container, login card
- `form` — all form wrappers (React Hook Form integration)
- `input` — text fields (NIT, razón social, prefix, passphrase)
- `label` — all field labels
- `progress` — wizard step progress bar
- `separator` — horizontal dividers between sections
- `badge` — environment indicator (HABILITACION / PRODUCCION)
- `alert` — blocking messages, error states
- `dialog` — confirmation prompt (not used in Phase 1, reserve for later phases)

**Source:** RESEARCH.md Standard Stack — Frontend; CONTEXT.md Claude's Discretion (shadcn/ui + Tailwind v4 is the stack).

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between tight elements |
| sm | 8px | Compact spacing: label-to-input, icon-to-text |
| md | 16px | Default spacing: form field gaps, card inner padding |
| lg | 24px | Section padding: wizard step content area, card padding |
| xl | 32px | Layout gaps: between major form sections |
| 2xl | 48px | Major section breaks: wizard header to content |
| 3xl | 64px | Page-level spacing: centered card top/bottom margin |

Exceptions:
- Touch targets for interactive controls (checkbox, radio, toggle): minimum 44px tall. This is not a spacing token — it is a minimum height constraint applied as `min-h-[44px]` via Tailwind utility.
- Wizard step indicator dots/numbers: 32px diameter (xl token used as size, not gap).

**Source:** 8-point scale default; touch target exception from accessibility standard (WCAG 2.5.5).

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (Regular) | 1.5 | `text-sm font-normal` |
| Label | 14px | 600 (Semibold) | 1.4 | `text-sm font-semibold` |
| Heading | 20px | 600 (Semibold) | 1.3 | `text-xl font-semibold` |
| Display | 28px | 600 (Semibold) | 1.2 | `text-2xl font-semibold` |

**Declared weights: 2 — 400 (Regular) and 600 (Semibold) only.**

**Usage notes:**
- Body (14px/400): form help text, secondary descriptions, metadata.
- Label (14px/600): all `<label>` elements, field hints, step subtitles. Elevated from 500 to 600 to stay within the 2-weight constraint.
- Heading (20px/600): wizard step titles ("Datos de la Empresa"), card headings.
- Display (28px/600): page-level title on login ("DIAN Facturador"), wizard main heading.

**Source:** Default for a functional B2B/professional tool. Sized for density without sacrificing legibility at body text.

## Color

shadcn/ui default neutral palette (slate-based). No custom color overrides for Phase 1 — install with defaults and apply semantic roles as follows.

| Role | Tailwind Semantic Token | Usage |
|------|------------------------|-------|
| Dominant (60%) | `background` / `bg-background` | Page background, main content areas |
| Secondary (30%) | `card` / `bg-card` or `muted` / `bg-muted` | Wizard container card, sidebar nav, step indicator background |
| Accent (10%) | `primary` / `bg-primary` | See reserved list below |
| Destructive | `destructive` / `bg-destructive` | Destructive actions only |

**Accent (primary) reserved for:**
1. "Guardar y continuar" and "Guardar" CTA buttons (wizard step submit buttons)
2. "Iniciar sesión" button on login page
3. Active wizard step indicator dot/circle
4. Environment badge when PRODUCCION is selected (to signal elevated risk)

**Accent (primary) NOT used for:**
- Informational text, body copy, secondary buttons, disabled states, links within prose.

**Environment badge color convention (critical — see RESEARCH.md Pitfall 7):**
- Entorno HABILITACION: `bg-yellow-100 text-yellow-800 border-yellow-300` — signals test mode.
- Entorno PRODUCCION: `bg-red-100 text-red-800 border-red-300` — signals live/real invoicing.
- Badge is displayed persistently in the app header/navbar at all times after setup completion.

**Destructive reserved for:**
- No destructive actions exist in Phase 1. Reserve token for future phases. If a "reset setup" or "delete resolution" action is added, use `bg-destructive`.

**Source:** CONTEXT.md "simple, approachable, not intimidating"; RESEARCH.md Pitfall 7 (environment must be displayed prominently).

## Component Inventory

### Page: LoginPage (`/login`)

| Component | Purpose | State |
|-----------|---------|-------|
| `Card` (center, max-w-sm) | Login form container | — |
| Display heading | "DIAN Facturador" | Static |
| Body text | "Ingresa para gestionar tu facturación electrónica" | Static |
| `Input` (type=text) | Usuario | Default, error |
| `Input` (type=password) | Contraseña | Default, error (with show/hide toggle via lucide Eye icon) |
| `Button` (primary, full-width) | "Iniciar sesión" | Default, loading (spinner + disabled), error |
| `Alert` (destructive) | Credenciales incorrectas | Shown only on auth failure |

**Interaction contract:**
- Form submit → button enters loading state (spinner) → on success redirect to `/onboarding` or `/` depending on setup status.
- On 401 error → show Alert below the form: "Usuario o contraseña incorrectos. Vuelve a intentarlo."
- Password field shows lucide `Eye` / `EyeOff` icon button at 16px, right-aligned inside input. Toggle button carries `aria-label="Mostrar contraseña"` when password is hidden and `aria-label="Ocultar contraseña"` when password is visible.

---

### Page: OnboardingPage (`/onboarding`)

**Layout:** Centered single-column card (max-w-lg) with step progress at top and step content below.

#### Wizard Structure

| Step | Title | Fields | Validation |
|------|-------|--------|------------|
| 1 | Datos de la Empresa | NIT (digits only), Razón Social, Régimen Fiscal (select), Dirección, Ciudad, Email, Teléfono (optional) | NIT required + check digit auto-calculated and shown; email format; all except phone required |
| 2 | Certificado Digital | File upload (.p12/.pfx), Contraseña del certificado | File type (.p12, .pfx), passphrase required; no DIAN validation (D-03) |
| 3 | Resolución de Facturación | Prefijo, Número desde, Número hasta, Clave técnica, Fecha inicio, Fecha vencimiento, Número de resolución (optional label) | All required except resolution number; date format; from < to number |
| 4 | Entorno DIAN | Radio select: Habilitación / Producción | Selection required; no default |

#### Step Progress Bar

| Element | Spec |
|---------|------|
| Container | `Progress` shadcn component, value = (currentStep / totalSteps) * 100 |
| Step labels | "Paso {N} de 4 — {step title}" in Label style |
| Active step dot | 32px circle with `bg-primary` |
| Completed step dot | 32px circle with lucide `Check` icon, `bg-primary/50` |
| Inactive step dot | 32px circle with `bg-muted` |

#### Per-step Interaction

- "Guardar y continuar" button: submits current step form → calls API → on success advances to next step.
- "Guardar y continuar" button loading state: spinner + disabled during API call.
- Step 4 final button label: "Completar configuración" (not "Guardar y continuar").
- No "Atrás" button — steps are not reversible during initial onboarding. User can revisit settings after setup in a future phase.
- On final step success: redirect to `/` (dashboard stub).

#### NIT Check Digit Display (Step 1)

- NIT input accepts digits only (no hyphens).
- Below the NIT input, display auto-calculated check digit: "Dígito de verificación: **{N}**" in Label style.
- Calculation triggers on blur (not on every keystroke).
- Full NIT format displayed as: `{NIT}-{check_digit}` in the display label.

#### Certificate Upload (Step 2)

| Element | Spec |
|---------|------|
| File drop zone | shadcn `Input` type=file with `accept=".p12,.pfx"`. No drag-and-drop (keep it simple for Phase 1). |
| File name display | After selection: show selected filename in muted text below input. |
| Passphrase field | `Input` type=password with show/hide toggle (lucide Eye/EyeOff). Toggle button carries `aria-label="Mostrar contraseña"` when password is hidden and `aria-label="Ocultar contraseña"` when password is visible. |
| Validation error | "Certificado inválido o contraseña incorrecta. Verifica que el archivo .p12 y la contraseña sean correctos." |

#### Entorno DIAN Selection (Step 4)

| Element | Spec |
|---------|------|
| Layout | Two `Card` options side by side (or stacked on mobile) acting as radio buttons |
| Habilitación option | Yellow accent card: "Habilitación (Pruebas)" + description "Para hacer pruebas ante la DIAN. Los documentos NO tienen validez legal." |
| Producción option | Red accent card: "Producción" + description "Para facturación electrónica real. Los documentos tienen validez legal ante la DIAN." |
| Selected state | `ring-2 ring-primary` border on selected card |
| No default | Neither card is pre-selected. "Completar configuración" button is disabled until a selection is made. |

## Responsive Behavior

Phase 1 targets desktop-first (the app is typically accessed from a PC/server where the user runs docker-compose). No mobile breakpoints are required for Phase 1. Minimum supported viewport: 1024px wide.

Exception: The layout must not break at 768px (tablet), but does not need to be optimized. Use Tailwind's `md:` breakpoint only if a component would otherwise overflow.

---

## Accessibility Baseline

- All form inputs have associated `<label>` elements (via shadcn `Form` + `FormLabel`).
- All error messages are associated with their input via `aria-describedby` (shadcn `FormMessage` handles this).
- Buttons in loading state are `disabled` and have `aria-busy="true"`.
- Color is never the sole indicator of state — environment indicator uses text label + color.
- Focus ring: use shadcn/ui default `focus-visible:ring` — do not remove it.
- Password show/hide icon buttons carry `aria-label="Mostrar contraseña"` (hidden state) and `aria-label="Ocultar contraseña"` (visible state). They are never icon-only without a label.

## Implementation Notes for Executor

1. **Tailwind v4 config:** No `tailwind.config.js` file. Import via `@import "tailwindcss"` in `src/index.css`. Use `@tailwindcss/vite` plugin in `vite.config.ts`.

2. **shadcn + Tailwind v4:** Use `npx shadcn@latest init` — the latest shadcn CLI supports Tailwind v4 natively. Do not follow older shadcn docs that reference PostCSS configuration.

3. **Environment badge placement:** Render the environment badge in the app shell (after login + setup). It must be visible on every page. Suggested location: top-right of the main nav bar, or in the sidebar header. Badge text and color follow the convention defined in the Color section above.

4. **No default environment (RESEARCH.md Pitfall 7):** The entorno field in the wizard Step 4 has no pre-selected value. The "Completar configuración" button is disabled until the user explicitly chooses.

5. **NIT formatting:** The NIT input accepts digits only. The check digit is displayed as a read-only computed value alongside the input. Do not combine NIT and check digit into a single field — keep them separate for clarity.

6. **Password show/hide toggle:** Use `lucide-react` `Eye` and `EyeOff` icons at 16px, placed as an absolute-positioned button inside the input's right padding area. Toggle the input `type` between `"password"` and `"text"`. Always set `aria-label` on the toggle button: "Mostrar contraseña" when the field is hidden, "Ocultar contraseña" when the field is visible.

7. **Step form submission flow:** Each step's form submits to its respective API endpoint. On success, the wizard advances. On error (4xx/5xx), the error is displayed in a `FormMessage` below the relevant field or in an `Alert` at the bottom of the card. The step does NOT advance on error.

8. **Loading states:** All CTA buttons use a spinner (lucide `Loader2` icon with `animate-spin` class) while the API call is in flight. The button text does not change — only the spinner is added as a prefix icon. Button is `disabled` during loading.
